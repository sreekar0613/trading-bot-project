## File Purposes

### Configuration Files
- **`.env`** — Stores API keys securely (you create this, never commit it)
- **`.gitignore`** — Prevents sensitive files from being pushed to git
- **`requirements.txt`** — Python package dependencies

### Documentation
- **`README.md`** — Overview, quick start, troubleshooting
- **`SETUP_GUIDE.md`** — Step-by-step installation (API accounts, environment setup)
- **`PHASE_1_PLAN.md`** — Claude Code prompts for data pipeline build
- **`.claude/CLAUDE.md`** — Full strategy specification (entry/exit logic, risk rules)

### API Skill Docs
- **`alpaca-api.md`** — Order execution, portfolio management, historical data
- **`finnhub-api.md`** — News sentiment, 7-day rolling averages
- **`alpha-vantage-api.md`** — Market cap, ROE, P/B ratios, earnings growth

### Security
- **`.git-hooks/pre-commit`** — Prevents committing API keys or `.env` file

## Files NOT in Repository (Created During Development)

### Generated by Phase 1 Scripts
- `trading_bot.db` — SQLite database (price history, fundamentals, sentiment)
- `*.py` scripts — Created by pasting Claude Code prompts
- `qualified_universe.csv` — List of stocks passing fundamental filters
- `data_quality_report.txt` — Missing data, gaps, validation issues

### Ignored by `.gitignore`
- `.env` — API keys
- `*.db` — Databases (cached data)
- `__pycache__/` — Python bytecode
- `venv/` — Virtual environment
- `*.log` — Log files

## Workflow: From Zero to Phase 1 Complete

1. **Download** → Unzip `trading-bot-starter-pack.tar.gz`
2. **Setup** → Follow `SETUP_GUIDE.md` to install dependencies and create API accounts
3. **Configure** → Create `.env` file with your API keys
4. **Validate** → Run test scripts to verify API connections
5. **Build** → Open `PHASE_1_PLAN.md`, paste prompts into Claude Code terminal
6. **Verify** → Share outputs with validation partner (me) after each prompt
7. **Checkpoint** → Review `qualified_universe.csv` before Phase 2
